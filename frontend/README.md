# HR-Buddy Frontend

Frontend web application for HR-Buddy, built with Next.js (App Router) and TypeScript.

This app serves 3 user groups:
- Employee/Public: create requests and track request status via OTP.
- Messenger: update messenger jobs via magic link token.
- Admin: manage requests, settings, audit logs, notifications, and attachments.

## Tech Stack

- Next.js 16
- React 19
- TypeScript
- Tailwind CSS v4

## Prerequisites

- Node.js 22+ (recommended)
- npm 10+
- HR-Buddy backend running (default `http://localhost:3001`)

## Environment Variables

Create `frontend/.env` from `frontend/.env.example`.

```ini
# Backend base URL used by frontend API client
NEXT_PUBLIC_API_BASE_URL=http://localhost:3001
```

Optional smoke tuning variables:

```ini
FRONTEND_SMOKE_BASE_URL=http://127.0.0.1:3000
FRONTEND_SMOKE_HOST=127.0.0.1
FRONTEND_SMOKE_PORT=3105
SMOKE_ROUTE_TIMEOUT_MS=15000
SMOKE_ROUTE_RETRIES=2
SMOKE_ROUTE_RETRY_DELAY_MS=500
SMOKE_SERVER_READY_TIMEOUT_MS=120000
SMOKE_API_TIMEOUT_MS=7000
SMOKE_REQUIRE_API=false
```

## Local Setup

```bash
cd frontend
npm ci
npm run dev
```

Open `http://localhost:3000`.

## Scripts

- `npm run dev`: start development server
- `npm run build`: build production bundle
- `npm run start`: start production server (requires build)
- `npm run lint`: ESLint check
- `npm run typecheck`: TypeScript check (`tsc --noEmit`)
- `npm run smoke`: route-level smoke test against existing frontend instance
- `npm run smoke:self-host`: start dev server and run smoke test
- `npm run smoke:self-host:prod`: start production server and run smoke test (requires build)
- `npm run smoke:strict`: smoke test + require backend health check
- `npm run release:gate`: lint + typecheck + build + smoke:self-host:prod
- `npm run release:gate:strict`: release gate with required backend health check

## Release Gate (No Deploy)

Before marking frontend as release-ready, run:

```bash
npm run release:gate:strict
```

This validates:
- Lint
- Type safety
- Production build
- Route smoke + backend health check

## Notes

- Auth/session tokens are stored in `sessionStorage` by default (per-tab persistence). Set `NEXT_PUBLIC_AUTH_TOKEN_STORAGE=memory` to disable browser persistence.
- Magic links generated by backend should point to `/messenger/link/:token`.
- This repository phase focuses on readiness and validation; deploy configuration is handled separately.
